Cuda Boid Sim

Jul 1, 2025

Hook

I wanted to see how GPU programming changes the way you think about algorithms. A flocking sim (Boids) was the perfect test case.

Background

Boid simulations model flocking behavior using simple rules: alignment, cohesion, and separation. They’re a great way to explore parallelism because each boid can be updated independently.
Insert GIF or screenshot of the Boid simulation here.

CPU Implementation

  • Traditional CPU approach: loops over each boid, updating positions and velocities.
  • Structure-of-arrays vs naive array-of-structs approach.
  • Limitations: performance drops with large numbers of boids.

CUDA Implementation

  • Programming style shift:
    • CPU: sequential loops, per-boid updates.
    • CUDA: kernels, blocks, threads, parallel execution.
  • Focus on avoiding data races and optimizing memory layout.
  • Tiny pseudocode contrast (optional).

Results & Visuals

  • Side-by-side demo: CPU vs CUDA at different scales.
  • Performance comparison:
    • CPU: ~30 FPS at 5k boids
    • CUDA: ~60 FPS at 50k boids
  • Let the visuals do the talking.

Reflections

  • Lessons learned about GPU programming:
    • Parallelism mindset
    • Debugging CUDA
    • Importance of memory layout
  • How this fits into broader learning goals: resume-worthy CUDA experience, real-time simulation work.
